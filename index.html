The OSI Model

The Open Systems Interconnection model is a conceptual framework used to describe the functions of a networking system.

Layer Name Function PDU Devices / Protocols
7 Application Network process to application Data HTTP, FTP, DNS, SMTP
6 Presentation Data representation and encryption Data SSL/TLS, JPEG, ASCII
5 Session Interhost communication (sessions) Data NetBIOS, PPTP
4 Transport End-to-end connections and reliability Segment TCP, UDP
3 Network Path determination and IP addressing Packet Routers, IPv4, IPv6, ICMP
2 Data Link MAC addressing and physical addressing Frame Switches, Bridges, MAC, ARP
1 Physical Media, signal, and binary transmission Bit Hubs, Cables (Cat6, Fiber), NICs

IP Addressing

Private IPv4 Ranges (RFC 1918)

  • Class A: 10.0.0.0 – 10.255.255.255 (/8)
  • Class B: 172.16.0.0 – 172.31.255.255 (/12)
  • Class C: 192.168.0.0 – 192.168.255.255 (/16)

Special Addresses

  • APIPA (Link-Local): 169.254.x.x (Failsafe when DHCP is unreachable)
  • Loopback (Localhost): 127.0.0.1 (IPv4) / ::1 (IPv6)

CompTIA Troubleshooting Methodology

  1. Identify the problem: Gather information, duplicate the problem, question users, identify symptoms.
  2. Establish a theory of probable cause: Question the obvious.
  3. Test the theory to determine cause: Once confirmed, determine the next steps to resolve. If not confirmed, establish a new theory.
  4. Establish a plan of action: Formulate a plan to resolve the problem and identify potential effects.
  5. Implement the solution or escalate: Do the fix, or send it to someone who can.
  6. Verify full system functionality: Apply preventative measures if applicable.
  7. Document findings, actions, and outcomes: Keep records for the future.
